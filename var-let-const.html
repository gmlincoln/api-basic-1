The scope of var, let, and const

In JavaScript, var,let and const keywords are used to declare variable. If var,let and const, three keywords are used for variable declaration, then what's the difference among them? We know, they have different types of scope and behaviors. Let's discuss about the differences among them.

1.var: If we declare variables with var keyword, then we have face some issues. Let's try to understand var.

Scope means where the variable are available for used. For var, it's declared as global scope, functional scope and block scope. When a variable declare with var keyword outside of a function, it is available for the entire scope. Again, if we declare a variable with var keyword inside a function, then variable only available inside the function.When we declare a variable with var keyword, We can re-declare and reassign value.

2.let: Now let is more preferred in variable declaration, we can reassign value inside it. Let's try to understand let.

let is block scope, that means it's working inside curly braces { }. So, if we declare a variable with let keyword, it is available inside the block. That why it is known as block scope. And we can't not redeclare a variable with let;
