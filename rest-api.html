What is REST API?

REST API stands for REpresentational State Transfer Application Programming Interface. A REST API also known as RESTful API. RESTful API is an interface use that two computer systems to exchange information between them securely over the internet. Lets talk about API and REST.
When we talk about REST API, a common question initially raised which is 'What is API'?

What is API?
An Application Programming Interface (API) establishes the guidelines and protocols for interacting with other software systems. Developers design and offer APIs to enable seamless programmatic communication between different applications. In API there are two portions, one is clients and another one is resources.
Lets talk about clients and resources.

Clients
Clients refer to individuals or software systems seeking access to data or services on the internet. The client can be a person or a software system that uses the API.
Resources
Resources are the information that applications are provide to their clients. Resources can be images, videos, text, numbers, or any type of data.

What is REST?
Representational State Transfer (REST) is a software architecture that defines a set of principles governing the functionality of an API. Originally conceived as a framework for managing communication over intricate networks like the internet, REST provides guidelines for designing efficient and scalable web APIs.

RESTful APIs are frequently constructed by developers using the Hypertext Transfer Protocol (HTTP). Within this context, HTTP methods instruct the server on how to interact with a resource. There are four prevalent HTTP methods: GET, PUT, POST,and DELETE.
